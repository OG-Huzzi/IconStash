Designing Accessible SVG Icons

A guide to structuring semantic code, configuring screen reader parameters, and implementing correct ARIA tags.

1. Hiding Decorative Vectors

The majority of interface icons are decorative, meaning they sit alongside text (like a shopping cart icon next to the text "Add to Cart"). For these icons, screen readers should skip them entirely to prevent vocal clutter.

To hide decorative vectors from assistive technologies, always apply the `aria-hidden="true"` attribute to the SVG tag:

<!-- Decorative icon hidden from screen readers -->
<svg viewBox="0 0 24 24" aria-hidden="true">
  <path d="..."/>
</svg>

2. Labeling Interactive Action Icons

When an icon is the only element inside a button or link (such as a search magnifying glass icon with no accompanying text), it acts as an interactive control. Screen readers need a clear label to convey its purpose.

Apply an `aria-label` or `title` to the wrapper button or link, and set `role="img"` on the SVG tag:

<button aria-label="Search database">
  <svg viewBox="0 0 24 24" role="img" aria-hidden="true">
    <path d="..."/>
  </svg>
</button>
